## Architecture

This is a Next.js 16 App Router application for AWS AIF-C01 exam prep, deployed to **Cloudflare Pages** (NOT Vercel). The `.vercel/` directory in the repo is a build artifact generated by Cloudflare's Next.js build tooling (via `@cloudflare/next-on-pages`), not evidence of Vercel hosting. Deployment is automatic on `git push` to `main` (GitHub repo connected to Cloudflare Pages). The app has a server-side endpoint for serving paid questions; everything else is client-side.

**Routing:**
- `/` â€” Home page with mode selection (Mock Test vs Flashcards)
- `/practice?mode=mock&feedback=end` â€” 65-question timed mock exam
- `/practice?mode=flashcard&feedback=immediate` â€” 10-question flashcard session
- `/resources/*` and `/help/*` â€” Static informational pages

**Core data flow:**
1. **Question pools** split into two non-overlapping sets:
   - `src/data/questions-free.json` â€” 30 questions, bundled client-side, powers flashcard mode
   - `src/data/questions-paid.json` â€” 179 questions, server-side only, powers mock-test mode via `/api/mock-questions`
2. **Flashcard mode:** `src/helpers/question-engine.ts` exports `getRandomQuestions` and `FREE_QUESTIONS` (bundled). Prioritizes unseen questions; `localStorage` key `aws-prep-seen-questions` tracks them.
3. **Mock-test mode:** `src/components/session/PracticeSession.tsx` fetches from `src/app/api/mock-questions/route.ts` (server endpoint), which selects 65 random paid questions and shuffles with Fisherâ€“Yates.
4. `src/context/SessionHistoryContext.tsx` persists seen question IDs to `localStorage`, used only by flashcard mode.
5. `src/components/session/PracticeSession.tsx` is the main stateful component â€” it owns the session state (questions, answers, bookmarks, timer) and orchestrates `QuestionCard`, `ResultsView`, `Sidebar`, and `SessionHeader`.

**Session modes:**
- `feedbackMode === "immediate"` (flashcard): answer revealed after each question, requires "CHECK ANSWER" button for multi-select
- `feedbackMode === "end"` (mock): answers revealed only after session finishes; sidebar nav and 90-minute countdown timer via `Sidebar`

**Question schema** (`Question` interface in `question-engine.ts`):
```ts
{ id, text, options, correctAnswer: string | string[], explanation, topic, multiSelect? }
```
Multi-select questions have `correctAnswer` as `string[]` and `multiSelect: true`.

**Styling:** Tailwind CSS v4 with `tailwind-merge` and `class-variance-authority`. Font is Plus Jakarta Sans via `next/font/google`. Color palette is light mode with indigo (`indigo-500`/`indigo-600`) as the primary accent.

**UI primitives:** `src/components/ui/button.tsx` and `card.tsx` are shadcn-style components using Radix UI slot and `cva`.

## Adding questions

Place raw questions in `new_questions.txt` (repo root) in the expected format, then run:

```bash
node scripts/import-questions.js
```

The script parses the format `Q: ... / Answer Choices / A. ... / Answer: X / Explanation ...`, deduplicates by normalized question text, and appends to `src/data/questions.json`. Other utility scripts: `scripts/add_single_question.js`, `scripts/remove_question.js`, and Python scripts in `src/data/` for processing/deduplication.
